Примеры использования системы (пополняется по мере разработки)

Здесь собраны примеры того, как можно использовать данную систему в Вашем бизнес-процессе и для информационной поддержки Ваших пользователей. Чтобы не загромождать это страницу QR-кодами, мы приведем здесь лишь несколько их примеров. В конце концов - QR-код это лишь удобный вспомогательный инструмент, чтобы быстро попасть со смартфона на нужную страницу.

1. Участник EXAMPLE

Участник имеет несколько диапазонов (16 диапазонов по 4096 объектов), настроенных для разных задач. Один из диапазонов он передал участнику EXAMPLED для построения и отладки примера использования системы, соответствующего реальной задаче, реального участника данного сервиса, компании Доминатор, которому данная система нужна для предоставления информации владельцам проданных ей судовых и индустриальных дизельных двигателей и приводов, таким образом, чтобы отсканировав QR-код на двигателе или в его тех-паспорте, владелец попал на страницу с характеристиками изделия, его гарантийным статусом и т.п.

Часть диапазонов зарезервирована им для будущего использования. Сейчас используются три:

1.2. 100A/XXX - здесь будет пример информации защищенной паролем

Пока не готово, поскольку никто из участников пока не попросил показать это функционал. Например это можно использовать для передачи бухгалтерских документов, или конструкторской документации, которую вы не хотите делать доступной всему миру. Пароль, естественно, можно поменять

  • Документ 1 - без пароля
  • Документ 1 - с паролем
  • Документ 2 - с паролем
  • Документ 2 - с неправильным паролем
  • ...
  • Пример QR-кода с паролем (Документ 1). Обратите внимание, насколько он сложнее, с точки зрения печати (требует больше места)

1.3. 100F/XXX - здесь будет пример перенаправления на другие сайты

Концепция состоит в том, что когда пользователь приходит в этот сервис, мы можем сразу перенаправить его на любую другую страницу в Internet, на которую укажет владелец данного объекта (участник системы). И когда ему потребуется изменить данное перенапрвление - это сделать очень просто, в отличии от переделывания всех QR-кодов, которые он уже разместил в большом количестве географически распределенных мест.
Вероятно, это будет самая востребованая модель использования.

  • Переадресация на сайт EustroSoft просто уведомление, с просьбой перейти на указаный сайт.
  • Переадресация на сайт EustroJAM перенаправление с задержкой в 10 секунд
  • Переадресация на сайт Доминатор безусловное перенаправление, пользователь может и не заметить, что заходил на наш сервис.
  • "Кроличья нора" - переадресация на случайный сайт из нашего списка интересных сайтов - пусть пока будет безусловная, QR-код прилагается ниже

2. Участник EXAMPLED

Данный пример, показывает как реально используется система для информационной поддержки пользователей проданного оборудования. Пример составляется на основе реального случая компании Доминатор, на основе данных о ее первых продажах, в далеком 2010 году. Данные слегка отредактированы, и отличаются от данных в собственной системе компании.

Итак, компания занимается поставкой судовых и индустриальных дизельных двигателей и приводов. Цель: отсканировав QR-код на двигателе или в его тех-паспорте, владелец должен попасть на страницу с характеристиками изделия, его гарантийным статусом и т.п.

  • Нулевой код 000 - будет указывать на информационную страницу обо всех объектах компании, с пояснениями.
  • Коды с 001 до 0FF (255 штук) - будут использованы для информации о самых первых проданных изделиях, с из серийными номерами, датой постановки на гарантию и датой окончания гарантии. Информация для отображения берется из простейшейго табличного файла (Excel), который есть у любого предпринимателя.
  • 001 Самый первый проданный двигатель
  • Коды с 100 до FFF - будут использованы для описаний модельного ряда изделий. По одному коду на одну модель.

    • 102 - TDME-2M78

    • 108 - TDME-490 (опять)

    • 10A - TDME-4105
    • 10B - TDME-4108
